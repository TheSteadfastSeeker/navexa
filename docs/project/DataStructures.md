# Data and Entity Structures: Predictive Analysis System for Shipping Machinery

## 1. Core Entities

| Entity | Description | Key Attributes | Relationships |
|--------|-------------|----------------|--------------|
| **Vessel** | Represents a ship or maritime vessel | • Vessel ID<br>• Name<br>• Type<br>• Build year<br>• Classification<br>• Dimensions<br>• Gross tonnage | • Has many Equipment<br>• Has many Voyages<br>• Belongs to Fleet<br>• Has many Maintenance Records |
| **Equipment** | Machinery or systems installed on vessels | • Equipment ID<br>• Type<br>• Manufacturer<br>• Model<br>• Installation date<br>• Specifications<br>• Location on vessel | • Belongs to Vessel<br>• Has many Sensors<br>• Has many Components<br>• Has many Maintenance Records |
| **Component** | Individual parts within equipment | • Component ID<br>• Name<br>• Type<br>• Manufacturer<br>• Serial number<br>• Installation date<br>• Expected lifetime | • Belongs to Equipment<br>• Has many Failure Modes<br>• Has many Maintenance Records |
| **Sensor** | Devices collecting data from equipment | • Sensor ID<br>• Type<br>• Location<br>• Measurement unit<br>• Calibration date<br>• Accuracy range<br>• Sampling frequency | • Belongs to Equipment<br>• Produces Sensor Readings<br>• Has Sensor Thresholds |
| **User** | System users with different roles | • User ID<br>• Name<br>• Role<br>• Permissions<br>• Contact info<br>• Organization<br>• Authentication details | • Belongs to Organization<br>• Has access to Vessels<br>• Creates Maintenance Tasks<br>• Receives Alerts |
| **Organization** | Companies operating vessels | • Organization ID<br>• Name<br>• Type<br>• Contact info<br>• Subscription level | • Has many Users<br>• Owns many Vessels<br>• Has many Fleets |
| **Fleet** | Logical grouping of vessels | • Fleet ID<br>• Name<br>• Type<br>• Description | • Belongs to Organization<br>• Contains many Vessels |

## 2. Operational Data

| Entity | Description | Key Attributes | Relationships |
|--------|-------------|----------------|--------------|
| **Sensor Reading** | Time-series data from sensors | • Reading ID<br>• Timestamp<br>• Value<br>• Quality indicator<br>• Collection method | • Belongs to Sensor<br>• Used in Anomaly Detection |
| **Operational State** | Current operating condition | • State ID<br>• Equipment ID<br>• Timestamp<br>• Operating mode<br>• Load percentage<br>• Environmental conditions | • Relates to Equipment<br>• Contextualizes Sensor Readings |
| **Voyage** | Journey information for vessels | • Voyage ID<br>• Start/end dates<br>• Route<br>• Cargo type<br>• Operating conditions<br>• Weather data | • Belongs to Vessel<br>• Contains many Operational States |
| **Maintenance Record** | Historical maintenance activities | • Record ID<br>• Date/time<br>• Type (planned/unplanned)<br>• Description<br>• Parts replaced<br>• Labor hours<br>• Cost | • Relates to Equipment or Component<br>• Created by User<br>• May be based on Maintenance Task |
| **Failure Event** | Recorded equipment failures | • Event ID<br>• Date/time<br>• Equipment/component affected<br>• Failure type<br>• Severity<br>• Impact<br>• Resolution | • Relates to Equipment or Component<br>• May relate to Failure Mode<br>• Results in Maintenance Records |

## 3. Analytical Models

| Entity | Description | Key Attributes | Relationships |
|--------|-------------|----------------|--------------|
| **Failure Mode** | Predefined ways equipment can fail | • Mode ID<br>• Name<br>• Description<br>• Typical indicators<br>• Typical causes<br>• Severity impact | • Relates to Equipment or Component<br>• Used in Prediction Models<br>• Referenced in Maintenance Procedures |
| **Prediction Model** | ML models for specific equipment | • Model ID<br>• Equipment type<br>• Trained date<br>• Version<br>• Accuracy metrics<br>• Input features<br>• Parameters | • Applied to Equipment type<br>• Generates Predictions<br>• Has Model Training History |
| **Model Training History** | Record of model training runs | • Training ID<br>• Model ID<br>• Date/time<br>• Dataset used<br>• Parameters<br>• Performance metrics | • Belongs to Prediction Model |
| **Prediction** | Generated forecasts for equipment | • Prediction ID<br>• Equipment ID<br>• Date generated<br>• Failure mode<br>• Probability<br>• Predicted timeframe<br>• Confidence score | • Generated by Prediction Model<br>• Relates to Equipment<br>• May generate Alerts<br>• May lead to Maintenance Tasks |
| **Anomaly Detection** | Unusual patterns in sensor data | • Anomaly ID<br>• Equipment ID<br>• Timestamp<br>• Type<br>• Severity<br>• Description<br>• Affected sensors | • Relates to Equipment<br>• Based on Sensor Readings<br>• May generate Alerts |
| **Remaining Useful Life** | Time until expected failure | • RUL ID<br>• Component ID<br>• Calculation date<br>• Estimated hours/days<br>• Confidence interval<br>• Methodology | • Relates to Component<br>• Used for Maintenance Planning |

## 4. Operational Management

| Entity | Description | Key Attributes | Relationships |
|--------|-------------|----------------|--------------|
| **Alert** | Notifications for users | • Alert ID<br>• Type<br>• Severity<br>• Timestamp<br>• Description<br>• Status<br>• Resolution notes | • Triggered by Prediction or Anomaly<br>• Sent to Users<br>• May create Maintenance Task |
| **Maintenance Task** | Scheduled work items | • Task ID<br>• Equipment ID<br>• Description<br>• Priority<br>• Status<br>• Scheduled date<br>• Estimated duration | • Relates to Equipment<br>• Assigned to Users<br>• May be based on Prediction<br>• Results in Maintenance Record |
| **Maintenance Procedure** | Standard procedures for tasks | • Procedure ID<br>• Equipment type<br>• Title<br>• Steps<br>• Required tools<br>• Required parts<br>• Safety precautions | • Applied to Equipment type<br>• Referenced in Maintenance Tasks |
| **Spare Part** | Inventory of replacement parts | • Part ID<br>• Name<br>• Compatible equipment<br>• Stock levels<br>• Supplier<br>• Cost<br>• Lead time | • Used in Maintenance Procedures<br>• Related to Components<br>• Used in Maintenance Records |
| **Sensor Threshold** | Boundary values for normal operation | • Threshold ID<br>• Sensor ID<br>• Min/max values<br>• Warning levels<br>• Context conditions | • Applies to Sensor<br>• Used in Anomaly Detection |

## 6. Data Storage Considerations

| Data Type | Storage Solution | Retention Policy | Backup Strategy |
|-----------|------------------|------------------|-----------------|
| **Time-series Sensor Data** | • Time-series database (InfluxDB/TimescaleDB)<br>• Partitioned by equipment and time periods | • Raw data: 30-90 days<br>• Aggregated data: 1-3 years<br>• Summary statistics: 5+ years | • Daily incremental<br>• Weekly full backup<br>• Geo-redundant storage |
| **Equipment Metadata** | • Relational database<br>• Document store for specifications | • Retained for life of system plus 5 years | • Daily full backup<br>• Point-in-time recovery |
| **Maintenance Records** | • Relational database<br>• Document store for attachments | • Retained for life of equipment plus regulatory period (typically 7+ years) | • Daily full backup<br>• Compliance-certified archiving |
| **Predictive Models** | • Model registry<br>• Version control system | • All versions retained for minimum 1 year<br>• Production versions indefinitely | • Version controlled<br>• Reproducible builds |
| **User Activity Data** | • Relational database<br>• Audit logs | • Detailed logs: 90 days<br>• Summary activity: 1 year<br>• Authentication records: 7 years | • Immutable audit trail<br>• Compliance-certified storage |
